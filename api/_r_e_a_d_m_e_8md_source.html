1 # Talvos
2 
3 This project provides a SPIR-V interpreter and Vulkan device emulator, with the
4 aim of providing an extensible dynamic analysis framework and debugger for
5 SPIR-V shaders.
6 Talvos provides an implementation of the Vulkan API to enable it to execute
7 real Vulkan applications.
8 Linux, macOS, and Windows are supported.
9

14 ## Status
15 
16 Talvos is still in the early stages of development.
17 Compute shaders are the current focus, and Talvos is currently capable of
18 executing various SPIR-V shaders generated from OpenCL kernels compiled with
19 [Clspv](https://github.com/google/clspv).
20 Talvos can also handle vertex and fragment shaders, with basic support for
21 offscreen rendering currently in progress.
22 Talvos currently passes around 15% of the
23 [Vulkan conformance test suite](https://github.com/KhronosGroup/VK-GL-CTS).
24 
25 The codebase is changing relatively quickly as new features are added, so the
26 internal and external APIs are all subject to change until we reach the 1.0
27 release (suggestions for improvements always welcome).
28 
29 Future work may involve extending the emulator to support tessellation and
30 geometry shaders, onscreen rendering, and implementing missing Vulkan API
31 functions to reach conformance.
32 Contributions in these (or other) areas would be extremely welcome.

35 ## Building
36 
37 More detailed build instructions are provided
38 [here](https://talvos.github.io/building.html).
39 
40 Building Talvos requires a compiler that supports C++17, and Python to enable
41 the internal test suite.
42 Talvos depends on
43 [SPIRV-Headers](https://github.com/KhronosGroup/SPIRV-Headers) and
44 [SPIRV-Tools](https://github.com/KhronosGroup/SPIRV-Tools), and uses GNU
45 readline to enhance the interactive debugger.
46 
47 Configure with CMake, indicating where SPIRV-Headers and SPIRV-Tools are
48 installed if necessary:
49 
50  cmake <path_to_talvos_source> \
51  -DCMAKE_BUILD_TYPE=Debug \
52  -DCMAKE_INSTALL_PREFIX=<target_install_directory> \
53  -DSPIRV_INCLUDE_DIR=<...> \
54  -DSPIRV_TOOLS_INCLUDE_DIR=<...> \
55  -DSPIRV_TOOLS_LIBRARY_DIR=<...>
56 
57 Using the `Debug` build type is strongly recommended while Talvos is still in
58 the early stages of development, to enable assertions that guard unimplemented
59 features.
60 
61 Once configured, use `make` (or your preferred build tool) to build and
62 install. Run `make test` to run the internal test suite.

65 ## Usage
66 
67 More detailed usage information is provided
68 [here](https://talvos.github.io/usage.html).
69 
70 Talvos provides an implementation of the Vulkan API which allows existing
71 Vulkan applications to be executed through the emulator without modification.
72 Simply linking an application against `libtalvos-vulkan.so` or
73 `talvos-vulkan.lib` is enough for it to use Talvos.
74 
75 Alternatively, the `talvos-cmd` command provides a simple interface to the
76 emulator.
77 An example that runs a simple N-Body simulation can be found in
78 `test/misc/nbody.tcf`.
79 
80 The following command is used to execute a Talvos command file:
81 
82  talvos-cmd nbody.tcf
83 
84 The command file contains commands that allocate and initialize storage buffers
85 and launch SPIR-V shaders.
86 There are also commands that dump the output of storage buffers, to verify
87 that the shader executed as expected.
88 
89 To enable the interactive debugger, set the environment variable
90 `TALVOS_INTERACTIVE=1`.
91 This will drop to a prompt when a shader begins executing.
92 Use `help` to list the available commands, which include `step` to advance
93 through the interpreter, `print` to view an instruction result, and `switch` to
94 change the current invocation.
